## Dynamic Type Status

32 instances of `.font(.system(size: N))` across 12 files — all use fixed point sizes.
The app also uses custom fonts `.a2zBold(size:)` and `.a2zRegular(size:)` which likely don't scale with Dynamic Type.

## Recommendations

1. **Add accessibility identifiers to all interactive elements** — buttons, NavigationLinks, and tappable calendar cells. This is prerequisite for any UI testing.
2. **Add accessibilityLabel to decorative images** — WorkoutIcon, Logo, chevrons for VoiceOver support.
3. **Create a UI test target** in Xcode and add `--uitesting` launch argument handling to the app.
4. **Consider Dynamic Type support** — replace hardcoded `.system(size:)` with semantic text styles (`.body`, `.caption`, etc.) for accessibility compliance. This is a larger effort.
